# GastApp Pulse design-sync notes

## Repo quirks

- `gastapp-ui` is **not a published npm package** — it is app-internal UI in `apps/web/src/components/ui/`. The converter cannot self-install it, so the build uses:
  - `--entry .design-sync/ui-entry.ts` — explicit TypeScript entry re-exporting all 8 primitives
  - `--node-modules .design-sync/node_modules` — synthetic node_modules dir with the gastapp-ui package (real `dist/*.d.ts` generated by `tsc -p .design-sync/tsconfig.ds.json`), Radix UI, react, and @types/react
  - `PKG_DIR` must resolve to `.design-sync/` (where `package.json` lives with `"name": "gastapp-ui"`) — do NOT move or rename `package.json` in `.design-sync/`

- **CSS**: Tailwind utilities are compiled by `buildCmd` via the Tailwind CLI into `.design-sync/compiled.css` before the converter runs. The raw `src/index.css` has `@tailwind` directives that esbuild cannot process — always run `buildCmd` first on re-sync.

- **@types/react location**: Installed in `.ds-sync/node_modules/@types/react` (by the `npm i esbuild ts-morph @types/react` step). Also symlinked/present in `apps/web/node_modules/@types/react`. The DTS scanner finds it via walk-up from `.design-sync/` → `.design-sync/node_modules/@types/react`.

- **Playwright render check**: The WSL2 host is missing `libasound2` (`sudo apt-get install libasound2`). Until installed, all validate runs need `--no-render-check`. Install playwright 1.42.x in `.ds-sync` (matches cached chromium-1105 build).

- **Dialog/ConfirmDialog previews**: These are Radix portal components — their overlays render into `document.body`. The previews use `open` prop with a fixed container so they appear inline. `cfg.overrides` sets `cardMode: "single"` and explicit viewports.

- **Sora font**: Loaded from Google Fonts via `index.html`. Marked as `runtimeFontPrefixes: ["Sora"]` — not shipped in the bundle, served at runtime by the host app.

## Re-sync risks

- **`compiled.css` is generated from the full web app source** — if new Tailwind classes are introduced in UI components, run `buildCmd` before re-syncing or they'll be missing.
- **`ui-entry.ts` must stay in sync** with the exports in `apps/web/src/components/ui/`. Adding a new component requires updating the entry file, re-running `tsc -p .design-sync/tsconfig.ds.json`, and copying the new `.d.ts` to `.design-sync/node_modules/gastapp-ui/dist/`.
- **`.design-sync/node_modules/gastapp-ui/dist/*.d.ts` are generated artifacts** — regenerate from `tsc` if component TypeScript APIs change.
- **Dialog/ConfirmDialog previews** are only viewable when Radix portals have a `document.body` — confirmed to compile fine; visual check needed in browser at `.review.html`.

## Known render warns

- `[RENDER_SKIPPED]` — missing `libasound2` on WSL2 host. Non-blocking; verified via browser-opened `.review.html`.
